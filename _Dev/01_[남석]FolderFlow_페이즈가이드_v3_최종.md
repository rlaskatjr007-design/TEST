## ⚠️ 공통 규칙 (매 페이즈 시작 전 확인)

- 각 페이즈는 **빌드 성공 상태**로 끝내야 다음 페이즈 진행
- 에러가 나면 같은 페이즈 안에서 해결하고 넘어갈 것
- 새 세션 시작할 때는 이 문서를 Claude Code에 다시 첨부할 것
- **테스트는 반드시 바탕화면에 만든 더미 폴더로 진행할 것** (실제 파일 건드리면 복구 어려움)

## Phase 0 — 프로젝트 세팅

### 📋 Claude Code 프롬프트

```
아래 조건으로 macOS 앱 프로젝트 기반을 세팅해줘.

조건:
- 앱 이름: Folder Flow
- 언어: Swift / SwiftUI
- 최소 지원 버전: macOS 13 이상
- 외부 라이브러리 없이 순수 SwiftUI만 사용
- 다크모드 비활성화 (라이트 모드 고정)
- 창 최소 크기: 900x600

파일 구조:
FolderFlow/
├── App/
│   └── FolderFlowApp.swift
├── Views/
│   ├── ContentView.swift
│   ├── SidebarView.swift
│   └── PanelView.swift
├── Models/
│   └── FileItem.swift
└── ViewModels/
    └── FileViewModel.swift

권한 설정:
- Info.plist에 파일 접근 권한 추가
  (NSDocumentsFolderUsageDescription, NSDesktopFolderUsageDescription)
- App Sandbox entitlements 설정
  (com.apple.security.files.user-selected.read-write)
- NSOpenPanel을 통한 파일 접근 권한 획득 방식 적용

각 파일은 빈 구조체로 만들어도 되고, 빌드만 성공하면 됨.
```

- [ ] 창 최소 크기 900x600 적용됨

## Phase 1 — 멀티 패널 + 사이드바

### 📋 Claude Code 프롬프트

```
Phase 0에서 만든 구조 위에 멀티 패널 기능을 구현해줘.

[사이드바]
- Finder처럼 즐겨찾기 섹션: 바탕화면, 다운로드, 문서, 홈 폴더
- iCloud Drive 섹션
- 외부 장치 섹션 (/Volumes 에서 마운트된 볼륨 자동 감지)
- 항목 클릭 시 현재 활성화된 패널에 해당 폴더 열기
  (활성화된 패널이 없으면 첫 번째 패널에 열기)

[패널 영역]
- 상단 분할 버튼: [1] [2] [3] [4]
- 버튼 클릭 시 패널 개수 즉시 변경 (균등 분할)
- 각 패널 상단에 현재 열린 폴더 경로 표시
- 파일/폴더 목록: 확장자 배지, 이름, 수정일, 크기 표시
- 폴더 클릭 시 해당 패널 안에서 하위 폴더로 이동

[패널 활성화]
- 패널 클릭 시 해당 패널 헤더에 색깔 강조 표시 (활성화 상태)
- 사이드바 폴더 클릭은 반드시 활성화된 패널에만 적용
- 순서대로 자동 지정되지 않음 — 클릭으로 활성화 후 사이드바 선택

[파일 열기]
- 파일 더블클릭 시 macOS 기본 앱으로 실행
  (이미지 → 미리보기, 영상 → QuickTime, 문서 → Pages/Acrobat 등)
- NSWorkspace.shared.open(_:) 사용

[드래그 앤 드롭]
- 패널 내 파일을 다른 패널로 드래그 가능
- 드롭 대상 패널에 초록 테두리 강조 표시
- FileManager를 사용해서 실제 파일 이동 처리
- 파일 접근 권한은 NSOpenPanel을 통해 획득
- 샌드박스 권한 관련 entitlements 설정 포함
- 드롭 완료 시 실제로 파일이 해당 폴더로 이동됨
- 이동 완료 후 두 패널 파일 목록 자동 새로고침
```

- [ ] 패널 클릭 시 헤더에 활성화 강조 표시가 됨
- [ ] 분할 버튼 1/2/3/4 클릭 시 패널 개수가 바뀜
- [ ] 패널 안에서 폴더 클릭 시 해당 폴더로 이동됨
- [ ] 파일 더블클릭 시 macOS 기본 앱으로 실행됨
- [ ] 파일을 다른 패널로 드래그하면 실제로 이동됨
- [ ] 드롭 시 초록 테두리 강조가 표시됨
- [ ] 이동 후 양쪽 패널 목록이 새로고침됨

## Phase 2 — 이미지 날짜별 자동 정리

### 📋 Claude Code 프롬프트

```
이미지 날짜별 자동 정리 기능을 구현해줘. 별도 탭 또는 시트로 열리면 됨.

[파일 불러오기]
- 상단 드롭존: 파일/폴더 드래그앤드롭으로 불러오기
- "폴더 선택" 버튼: NSOpenPanel로 폴더 선택
- 두 방법 모두 지원

[정리 방식 선택 — 사용자가 선택 가능]
정리 시작 전 아래 두 가지 방식 중 선택:

옵션 A — 이미지 폴더 안에 날짜별 정리
  선택한 폴더/
  └── 이미지/        ← 없을 경우에만 새로 생성
      ├── 2026-03-25/
      ├── 2026-03-20/
      └── 2026-02-14/

옵션 B — 날짜 폴더만 바로 생성
  선택한 폴더/
  ├── 2026-03-25/
  ├── 2026-03-20/
  └── 2026-02-14/

[날짜 폴더 중복 처리 규칙 — 중요]
- 해당 날짜 폴더가 이미 존재하는 경우 → 폴더 새로 만들지 않고 기존 폴더에 파일 추가
- 같은 날 여러 번 정리해도 날짜 폴더는 하나만 유지
- 이미지 폴더(옵션 A)도 이미 존재하면 새로 만들지 않고 기존 폴더 사용

[미리보기]
- 정리 시작 전 날짜별 트리 구조 미리보기 표시
- 새로 생성될 폴더와 기존 폴더에 병합될 폴더 구분 표시
  - 새로 생성: [새 폴더] 배지
  - 기존 폴더에 추가: [기존 폴더에 추가] 배지
- 이미지가 없는 날짜는 폴더 생성 안 함

[정리 실행]
- 이미지 파일(.jpg .jpeg .png .gif .heic .webp)만 추출
- 파일의 생성일 기준으로 날짜 그룹 분류
- "정리 시작" 버튼 클릭 시 실제로 폴더 생성 및 파일 이동
- 정리 완료 후 완료 메시지 표시

[공통]
- 정리 전 반드시 미리보기 단계 거칠 것
```

### ✅ 점검 항목

- [ ] 폴더 드래그앤드롭으로 선택 가능
- [ ] "폴더 선택" 버튼으로 선택 가능
- [ ] 옵션 A / 옵션 B 선택 UI가 있음
- [ ] 이미지 파일만 추출됨 (다른 파일 제외 확인)
- [ ] 날짜별 트리 미리보기가 정리 전에 표시됨
- [ ] 새 폴더 / 기존 폴더에 추가 배지가 구분 표시됨
- [ ] 이미지 없는 날짜는 폴더가 생성되지 않음
- [ ] 이미 존재하는 날짜 폴더에는 파일만 추가됨 (폴더 중복 생성 안 됨)
- [ ] 이미 존재하는 이미지 폴더도 중복 생성 안 됨 (옵션 A)
- [ ] 정리 시작 후 실제로 날짜별 폴더가 만들어짐
- [ ] 파일이 올바른 날짜 폴더로 이동됨
- [ ] 완료 메시지 표시됨
- [ ] 빌드 성공 (에러 0개)

## Phase 3 — 파일 이름 일괄 변경

### 📋 Claude Code 프롬프트

```
파일 이름 변경 기능을 구현해줘. 별도 탭 또는 시트로 열리면 됨.

[파일 불러오기]
- 상단 드롭존: 파일/폴더 드래그앤드롭으로 불러오기
- "폴더 선택" 버튼: NSOpenPanel로 폴더 선택
- 두 방법 모두 지원

[모드 A — 일괄 넘버링]
- 폴더 또는 파일 여러 개 선택
- 기본 이름 입력 필드 제공
- 입력하는 즉시 미리보기 실시간 업데이트
  예) "가맹점사진" 입력 → 가맹점사진_001.jpg / 가맹점사진_002.jpg / 가맹점사진_003.jpg
- 시작 번호 지정 가능
- 오른쪽 영역을 넓게 활용: 미리보기 목록을 크고 명확하게 표시

[모드 B — 개별 이름 변경]
- 좌측: 파일 목록 (확장자 배지 + 원본명 + 변경될 이름 미리보기 실시간 반영)
- 우측: 각 파일마다 이름 직접 입력 필드
  예) 1번: A가맹점 / 2번: B가맹점 / 3번: C가맹점
- 입력할 때마다 좌측 미리보기 실시간 반영

[저장 방식 — 두 모드 공통]
- 바로 저장: 원본 파일 이름 즉시 변경
- 별도 저장: 다른 경로에 이름 변경된 복사본 저장 (NSOpenPanel로 경로 선택)

[공통]
- 확장자는 자동 유지 (사용자가 입력 안 해도 됨)
- 변경 전 미리보기 제공
```

### ✅ 점검 항목

- [ ] 폴더 드래그앤드롭으로 파일 불러오기 가능
- [ ] "폴더 선택" 버튼으로 파일 불러오기 가능
- [ ] 모드 A/B 전환 버튼이 있음
- [ ] 모드 A: 기본 이름 입력 시 미리보기가 실시간으로 바뀜
- [ ] 모드 A: 오른쪽 영역에 미리보기 목록이 크게 표시됨
- [ ] 모드 A: 바로 저장 클릭 시 실제 파일 이름이 바뀜
- [ ] 모드 A: 별도 저장 클릭 시 경로 선택 후 복사본 저장됨
- [ ] 모드 B: 파일마다 개별 입력 필드가 있음
- [ ] 모드 B: 입력 시 좌측 미리보기가 실시간 반영됨
- [ ] 모드 B: 바로 저장 / 별도 저장 동작함
- [ ] 확장자가 자동으로 유지됨

## Phase 4 — 압축 파일 삭제

### 📋 Claude Code 프롬프트

```
특정 폴더 안의 압축 파일만 찾아서 삭제하는 기능을 구현해줘. 별도 탭으로 열리면 됨.

[파일 불러오기]
- 상단 폴더 경로 지정 필드
- 드래그앤드롭 또는 "폴더 선택" 버튼으로 지정
- 두 방법 모두 지원

[기능]
- 지정한 폴더 안의 압축 파일만 스캔
  지원 확장자: .zip .rar .7z .tar .gz
  (.alz 제외 — macOS 기본 지원 안 함)
- 파일 목록 표시: 확장자 배지, 파일명, 크기, 수정일
- 각 파일 왼쪽에 체크박스 제공
- "전체 선택" / "전체 해제" 버튼
- 우측 요약 패널: 발견된 파일 수, 선택된 수, 삭제 예정 용량
- "선택 항목 삭제" 버튼 클릭 시 휴지통으로 이동 (즉시 삭제 아님)
- 삭제 완료 후 목록 자동 새로고침

[주의]
- 데스크탑 전체나 시스템 폴더는 절대 건드리지 말 것
- 반드시 사용자가 지정한 폴더만 스캔
```

### ✅ 점검 항목

- [ ] 폴더 경로 지정이 드래그 또는 버튼으로 가능
- [ ] 지정한 폴더 안의 압축 파일만 목록에 표시됨
- [ ] .alz 확장자는 목록에 표시되지 않음
- [ ] 체크박스로 개별 선택/해제 가능
- [ ] 전체 선택 / 전체 해제 버튼 작동
- [ ] 우측에 요약 정보 (파일 수, 선택 수, 용량) 표시됨
- [ ] 삭제 시 즉시 삭제가 아닌 휴지통으로 이동됨
- [ ] 삭제 후 목록이 자동으로 새로고침됨
- [ ] 빌드 성공 (에러 0개)

```

  - 🖼 이미지 정리 (Phase 2)
  - ✏️ 이름 변경 (Phase 3)
  - 🗜 압축 파일 삭제 (Phase 4)

  - 우측: 설정, 옵션, 미리보기, 요약 정보
- 각 페이지 상단에 드롭존 + 폴더 선택 버튼 공통 제공

[디자인 통일]
- 전체 폰트, 색상, 버튼 스타일 통일
- 라이트 모드 일관성 확인
- 여백, 패딩 정리

[기타]
- 앱 아이콘 기본 설정 (SF Symbol 활용 가능)
- 앱 이름 "Folder Flow" 타이틀바에 표시
- 창 최소 크기 900x600 재확인
```
